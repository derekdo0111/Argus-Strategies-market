## 输出要求

只输出一个 JSON 数组，每个元素**仅有 4 个字段**：

- id: H{{层级}}-{{序号}}（如 H0-1, H1-1, H2-1, H3-1）
- title: 简短标题（≤10字）
- chain_level: 0/1/2/3
- derives_from: 字符串数组（L0 为空数组 []）

## 层级约束

| 层级 | 名称 | 数量范围 | derives_from 规则 |
|------|------|:---:|------|
| L0 | 现状诊断 | 2-3条 | 空数组 [] |
| L1 | 一阶推演 | 2-4条 | 每条引用 ≥1 个 L0 的 id |
| L2 | 二阶推演 | 2-4条 | 每条引用 ≥1 个 L1 的 id |
| L3 | 投资落点 | 2-3条 | 每条引用 ≥1 个 L2 的 id |

## 链完整性硬约束（不可违反）

1. 每条 L0 必须至少产生 1 条 L1（可从多条 L0 合并导出）
2. 每条 L1 必须至少产生 1 条 L2
3. 每条 L2 必须至少产生 1 条 L3
4. 不允许死胡同 — 如果某条 L1 确实无法推导出 L2，合并到其他链条或删除

## 输出格式

只输出 JSON，不要任何解释文字：

```json
[
  {{"id": "H0-1", "title": "AI算力需求爆发", "chain_level": 0, "derives_from": []}},
  {{"id": "H1-1", "title": "AI基建投资扩张", "chain_level": 1, "derives_from": ["H0-1", "H0-2"]}},
  {{"id": "H3-1", "title": "聚焦算力核心设备", "chain_level": 3, "derives_from": ["H2-1"]}}
]
```
