# 2025-06-15

## 修复：选股器从 9 只 → 50 只（fina_indicator 批次截断 bug）

### 问题
选股器 5529 只只通过了 9 只，最大淘汰项 ROE<=0 淘汰了 2332 只（42.2%）。

### 诊断
- `fina_indicator` Tushare API 单次调用最多处理约 100 只股票，超出静默截断
- 原代码 `_fetch_all_fina_indicator(batch_size=500)` → 每次只拿到 ~20% 数据 → fillna(0) → 误淘汰
- 验证：batch=50 返回率 100%，batch=100 返回率 88%，batch=200 返回率 47%

### 修复
- `backend/app/services/data_fetcher.py` `_fetch_all_fina_indicator`: batch_size 500 → 60
- 添加覆盖率日志 `fina_indicator 拉取完成: 5525/5529 只 (99.9%)`
- 添加截断告警（当返回率低于 80% 时）
- 修复 `scripts/run_turtle_refresh.py` sys.path 缺少 backend/

### 结果
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| fina 覆盖率 | 30.5% | 99.9% |
| ROE<=0 淘汰 | 2332 | 489 |
| 通过数 | 9 | 50 |
| 通过股示例 | - | 泸州老窖、分众传媒、东方财富、巨人网络... |
