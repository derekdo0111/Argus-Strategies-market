- Run1: 高景气 h=12 stocks=0 signal=3.52 | 4 confirmed + 4 disputed + 4 unreachable
- Run2: 高景气 h=14 stocks=0 signal=3.34 | 3 confirmed + 6 overturned + 1 partial + 4 unreachable
- Run3: 景气   h=14 stocks=23 signal=2.67 | 3 confirmed + 3 disputed + 2 overturned + 1 partial + 4 unverified + 1 unreachable

**新增问题**：
1. HypothesizeAgent 输出不同（Run1=12假设, Run2/Run3=14假设）— Run2 凭空多出 H2-4 "国产替代加速" 和 H3-4 "配置国产替代龙头"
2. CounterAgent 级联放大：Run1 disputed→Run2 变成了 overturned（6个），Run3 又部分回 disputed
3. ScreeningAgent 极端波动：0→0→23 stocks（same hypotheses 内容，不同 status 导致 23 stocks 全或无）
4. verified_sentiment 也不稳定：H2-2 "HBM胜出" 的 verified_sentiment 在 3 次 outer run 间是 positive→negative→positive

**Tavily 缓存 100% 命中**：14/14 hits，0 misses，外部输入冻结确认有效。

**根因分析**：3 轮投票只修了 VerifyAgent 内部的稳定性，但 HypothesizeAgent（假设生成）、CounterAgent（级联安全网）、ScreeningAgent（选股）都是单次 LLM 调用，未被稳定化。

**级联路径**：
```
Hypothesize 不同假设集（12 vs 14）
  → Verify 不同链条验证（H3-4 缺失 vs 存在）
  → CounterAgent 不同 overturn 决策（6 vs 2）
  → Screening 不同选股（0 vs 23）
  → rating 从高景气跳景气
```

**下一步建议**：
- P0: HypothesizeAgent 也加 self-consistency（假设数量主因）
- P1: CounterAgent 软衰减（防 overturn 过度）
- P2: Screening 结果不依赖 hypothesis status 的精确值
