## 稳定性实测：人工智能板块两遍对比 — 20/100 分（差）

### 执行
- 脚本: `scripts/stability_compare.py`（新建，可复用，支持 `--no-run` 只对比模式）
- 命令: `python scripts/stability_compare.py 人工智能`
- Run 1: 603s (10.1min), session 36, 评级「景气」
- Run 2: 431s (7.2min), session 37, 评级「高景气」

### 结果

| 维度 | Run 1 | Run 2 | 一致? |
|------|-------|-------|-------|
| confirmed | 2 | **8** | ❌ 差 4 倍 |
| partial | 6 | 4 | ❌ |
| disputed | 1 | 1 | ✅ |
| unverified | 4 | 0 | ❌ |
| 股池大小 | 23 | 27 | ❌ |
| 纯度分差异 | — | — | ❌ 16/23 只 |
| 排名变化 | — | — | ❌ 20/23 只 |
| 评级 | 景气 | 高景气 | ❌ |

### 根因
- `_synthesize_status()` 是确定性的 ✅，但 3 个输入 (source_count/data_alignment/counter_conflict) 仍是 LLM 输出，在两次运行间剧烈漂移
- 关键词纯度分依赖 L3 假设的 investment_implication 文本 → 假设变化 → 关键词不同 → 纯度差异
- 全链路 LLM 波动叠加：Search(Tavily) → Hypothesize → Verify → Screening

### 结论
确定性改造只解决了「合成」环节，未解决「输入」环节。需要 Search 冻结 + 假设固定化才能根治。
