## 高景气策略推理链路 3 层 Bug 修复 (v0.18.6)

### 问题发现
通过 brainstorming 发现高景气策略推理链中 3 个结构性问题：
1. **P0: Polarity-blind cascade** — `_cascade_safety_net` 仅看 status 不看 sentiment，negative+disputed（证明行业健康）被错误截断
2. **P1: Chain C 死端** — HypothesizeAgent 允许 L1 无 L2、L2 无 L3，导致链条结构性不完整
3. **P2: SIGNAL_MAP 仅 5/9 覆盖** — 4 种组合默认 0 分，且双重惩罚已打折信号

### 根因
DISPUTED ≠ OVERTURNED。原始 Spec §2.4 定义两阶段：VerifyAgent→DISPUTED→CounterAgent→OVERTURNED→截断。CounterAgent 被删除后 `_cascade_safety_net` 将 DISPUTED 当终端态处理。

### 修复方案（3 层）

#### Layer 1: CounterAgent 语义级联 + Sentiment 修正 (Plan B)
- **新文件**: `counter_agent.py` (~280 行)
  - `_llm_cascade()`: LLM 读取 5 维验证输出 (status/sentiment/reason/causality_strength/corrected_statement) → 极性感知识别
  - `_hardcoded_polarity_cascade()`: LLM 失败时 keyword-based sentiment correction + `disputed+negative→keep alive`
  - `_cascade_safety_net`: 保留作为第三级 fallback
- **3 级降级**: LLM cascade → hardcoded polarity rules → original safety net
- **Sentiment 修正 Plan B**: CounterAgent 直接覆盖 `h["sentiment"]`（保留 `original_sentiment` 审计）→ ReportAgent SIGNAL_MAP 自动消费

#### Layer 2: Hypothesize 结构完整性约束
- Prompt: Rule 8 — 每个 L1 至少 1 个 L2，每个 L2 至少 1 个 L3
- Code: `_validate_chain_completeness()` 软约束（warning 不阻塞）

#### Layer 3: SIGNAL_MAP 全覆盖
- 从 5→12 条目，覆盖全部 12 种 (sentiment×status) 组合
- 移除 `DEFAULT_SIGNAL=0.0` 和双重惩罚 `×0.5`

### 改动文件（10 个）

## Pipeline 输出修复 + LLM 超时增强 (v0.18.9-dev)

用户反馈「最后一步总是失败」，经排查发现 3 个独立问题：
1. **Step 编号不一致** — `1/5`、`2/5`、`5/5` 未随 CounterAgent 新增更新为 `1/6`、`2/6`、`6/6`

- `verify_agent.py`: timeout 120→180s，新增 `requests.exceptions.Timeout` 重试一次
