### v1.1.0 假设树感知分赛道筛选 + 验证诊断展示（Phase 7）

- **背景**: 用户详细 review 了 Session 139 存储芯片的完整输出，发现两个核心问题：
  1. 股池 170 只混在一张表按纯度排名——分销商排第一、锂电设备混进存储芯池、北方华创标成 H3-1 实应 H3-2——读者分不清核心标的 vs 蹭概念
  2. 合成报告看不到修正陈述（如 H2-1 "产能过剩隐忧" 被修正为 "供应紧张格局难以逆转"），验证环节核心发现被丢弃
- **讨论过程**: A+C 结合方案（产业链环节结构性分组 + 整棵假设树消费）→ 一条假设都不能硬编码关键词（跨行业维护不了）→ 修正过的假设不能扣分（causality=broken→weight=0）→ LLM 不能省但要换任务
- **新建**: `tools/screening_scorer.py`（~190行）
  - `hypothesis_weight(h)`: causality=broken→0, overturned/unreachable→0, 有修正→×0.5
  - `format_hypothesis_tree()`: 假设树→LLM prompt（正面/负面/中性分组，broken 标注不参与判断）
  - `score_stock_pool()`: 三维打分（景气适配 Σ正面命中×权重, 风险暴露 Σ负面命中×权重, 质量归一化）
- **重写**: `screening_agent.py`（4阶段→3阶段）
  - Stage 1: `_llm_classify_and_match()` — LLM 单次调用：分类到环节+标记命中了哪些假设+关联占比(0~1)
  - Stage 2: 程序化三维打分（消费 scoring_scorer）
  - Stage 3: 财务拉取（仅存活股票）→ 分赛道排名
  - 删旧 `_llm_direction_match` + `_compute_purity`（合并到 Stage 1 LLM 中）
  - 新增 `_programmatic_classify()` LLM 不可用时关键词兜底
- **重写**: `screening_direction_prompt.md` — 任务从「打分0~1」→「分类+标记命中+关联占比」
- **修改**: `report_agent.py` — 每条假设加「验证诊断」(因果链强度+修正陈述+验证说明)；股池拆三张分段表(上游/中游/下游)
- **测试**: 208/208 ✅ 零回归（修复 2 个旧测试适配新表头）
- **版本**: v1.0.5 → v1.1.0
