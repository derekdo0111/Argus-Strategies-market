# 2026-07-06 工作日志

## LLM 超时快速失败机制 (v0.21.2)

### 背景
07-05 收敛性实验 3 次全超时，每轮等待 7.5 分钟才退出——总计浪费 22+ 分钟。根因分析：
1. `_call_llm_phase1` 硬编码 30s 超时无视 `.env` 配置（45s）
2. 超时被 `_call_llm` 吞掉返回 `"[]"`，触发多层 fallback 重试链
3. 腾讯云代理 `deepseek-v4-pro` 响应慢

### 修改内容

**hypothesize_agent.py**：
- 新增 `LLMUnavailableError` 异常类
- `_call_llm`: Timeout/ConnectionError → `raise LLMUnavailableError`（不再吞异常）
- `_call_llm_phase1`: 硬编码 `30` → `getattr(settings, "PROSPERITY_HYPOTHESIZE_PHASE1_TIMEOUT", 30)`
- `_phase1_skeleton`: 全轮超时 → `raise LLMUnavailableError`（不再 fallback 到 120s 单轮）
- `_phase2_fill`: 超时 → `raise LLMUnavailableError`（不再重试 3×60s）
- `form_hypotheses`: Phase 2 LLM 不可用时 propagate 异常

**convergence_experiment.py**：
- `run_pipeline_with_patches`: 捕获 `LLMUnavailableError`，返回 `{"status": "failed"}`
- `main()`: 每个 Run 后检查 `status == "failed"`，立即终止实验

**.env**：
- `LLM_MODEL`: `deepseek-v4-pro-202606` → `deepseek-v4-flash`（更快更稳定）

### 效果
| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| Phase 1 全超时 | ~450s | 上限 45s 后立即退出 |
| Phase 2 超时 | ~300s | 60s 后立即退出 |
