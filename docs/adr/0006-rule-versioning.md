# ADR-0006: 规则版本化

- **日期**: 2026-06-15
- **状态**: 已接受

## 上下文

策略的选股条件、门控阈值等规则会随策略迭代而变化。需要追踪规则变更历史，并保证历史计算结果可复现。

## 决策

采用规则版本化方案：
- 规则存储在 `rules/v{N}/` 目录下
- 每个版本目录包含完整的规则文件（YAML）
- 数据库记录计算结果时附带规则版本号
- 规则版本号遵循语义化版本：`v{major}.{minor}`

示例结构：
```
rules/
├── v1/
│   ├── turtle_screener.yaml      # 选股条件
│   ├── turtle_cash_quality.yaml  # 现金质量门阈值
│   ├── turtle_pr.yaml            # PR 计算公式和阈值
│   └── turtle_fundamental.yaml   # 基本面分析配置
```

## 后果

- 规则变更可追溯
- 历史计算结果可复现（用对应版本重跑）
- 需要维护版本迁移逻辑（规则变更时的兼容性处理）
